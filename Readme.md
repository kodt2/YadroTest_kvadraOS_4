# Media Scanner

`media_scanner` — приложение на C++ для Linux, которое периодически обходит каталог с файлами, находит мультимедийные файлы и отдает результат в формате JSON через HTTP.

По умолчанию HTTP-результат доступен по адресу:

```text
http://localhost:1234/media_files
```

JSON имеет вид:

```json
{
  "audio": ["111.mp3", "222.wav"],
  "video": ["333.mpg"],
  "images": ["444.jpeg", "555.png"]
}
```

## Возможности

- рекурсивный обход выбранного каталога;
- поиск аудио-, видео- и графических файлов по расширениям;
- периодическое обновление результата через заданный интервал времени;
- настройка каталога сканирования через аргумент запуска;
- настройка интервала сканирования через аргумент запуска;
- настройка списка расширений через `config.json`;
- выдача результата через HTTP GET-запрос.

## Требования

- Linux;
- компилятор с поддержкой C++17, например `g++`;
- CMake версии 3.14 или выше;
- доступ к интернету во время первой сборки, так как зависимости загружаются через `FetchContent`:
  - `nlohmann/json`;
  - `cpp-httplib`.

## Сборка

Из корня репозитория выполните:

```bash
cmake -S . -B build
cmake --build build
```

После успешной сборки исполняемый файл будет находиться по пути:

```text
build/media_scanner
```

## Запуск

Базовый запуск из корня репозитория:

```bash
./build/media_scanner
```

После запуска приложение начинает периодически сканировать каталог и поднимает HTTP-сервер на порту `1234`.

Проверить результат можно командой:

```bash
curl http://localhost:1234/media_files
```

Или открыть адрес в браузере:

```text
http://localhost:1234/media_files
```

## Аргументы командной строки

Приложение поддерживает следующие аргументы:

| Аргумент | Описание | Пример |
|---|---|---|
| `--config` | путь к JSON-файлу конфигурации с расширениями файлов | `--config config.json` |
| `--target` | путь к каталогу, который нужно сканировать | `--target /home/user` |
| `--interval` | интервал сканирования в миллисекундах | `--interval 5000` |

Пример запуска с явным указанием всех параметров:

```bash
./build/media_scanner --config config.json --target "$HOME" --interval 5000
```

Пример запуска для тестового каталога:

```bash
mkdir -p /tmp/media_test
touch /tmp/media_test/111.mp3 /tmp/media_test/222.wav /tmp/media_test/333.mpg /tmp/media_test/444.jpeg
./build/media_scanner --config config.json --target /tmp/media_test --interval 1000
```

В другом терминале можно проверить HTTP-ответ:

```bash
curl http://localhost:1234/media_files
```

Ожидаемый ответ будет похож на:

```json
{"audio":["111.mp3","222.wav"],"images":["444.jpeg"],"video":["333.mpg"]}
```

## Конфигурация расширений

Список расширений задается в файле `config.json`:

```json
{
  "extensions": {
    "audio": [".mp3", ".wav", ".ogg"],
    "video": [".mp4", ".mkv", ".avi"],
    "images": [".jpg", ".jpeg", ".png"]
  }
}
```

В проектном `config.json` уже указан расширенный список популярных аудио-, видео- и графических форматов.

Расширения указываются с точкой, например `.mp3`, `.png`, `.mp4`.

## Примечания

- Сервер запускается на порту `1234`.
- Результат обновляется периодически, а не только один раз при старте.
- Если каталог содержит файлы с неподдерживаемыми расширениями, они игнорируются.
- Расширения файлов сравниваются без учета регистра: например, `.MP3` будет обработан как `.mp3`.
